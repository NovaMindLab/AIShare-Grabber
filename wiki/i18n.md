## 🌍 Web Portal (Vue 3 + Vite)

### Architecture

The web localization system is defined in [`cp_clip/src/locales.js`](file:///d:/AI_serach_image/image_clip_android/cp_clip/src/locales.js).

- **`locales`** — A plain JS object keyed by locale code (e.g. `en`, `zh`, `fr`). Each locale key maps to a flat dictionary of UI strings.
- **`languages`** — A `Map` of locale code → display name, used to populate the language-picker dropdown.

### Usage in Vue components

```js
import { locales, languages } from './locales.js';

const currentLocale = ref('en');  // ← defaults to English
const t = computed(() => locales[currentLocale.value] || locales.en);

// Translate a key in the template:
// {{ t.hero.title }}
// {{ t.features.downloadEXE }}
```

### Language Picker

A sticky language selector is rendered in the navigation bar. Clicking a language code instantly swaps all rendered strings without a page reload (fully reactive via Vue computed properties). The selection is **not** persisted across sessions for the web portal (stateless public page).

## 🖥️ PC Desktop Client (Electron + Vue 3)

### Architecture

The desktop localization system is defined in [`cp_clip/src/locales.js`](file:///d:/AI_serach_image/image_clip_android/cp_clip/src/locales.js) — the **same file** as the web portal.

The Electron renderer imports `locales` and `languages` directly:

```js
import { locales, languages } from './locales.js';

const currentLocale = ref('en');   // ← defaults to English
const t = computed(() => locales[currentLocale.value] || locales.en);
```

### Persistence

The selected language is saved to **`localStorage`** so the choice is remembered across app restarts:

```js
// Save on change
localStorage.setItem('shareclip_locale', langCode);

// Restore on mount
onMounted(() => {
  const saved = localStorage.getItem('shareclip_locale');
  if (saved && locales[saved]) currentLocale.value = saved;
});
```

### Translatable UI Regions

| UI Area                    | Key Namespace Example         |
|----------------------------|-------------------------------|
| Sidebar tab labels         | `t.tabs.images`, `t.tabs.videos` |
| Link tab placeholder       | `t.link.placeholder`          |
| Image detail panel         | `t.details.imageName`, `t.details.category` |
| AI analysis captions       | `t.images.aiAnalyzing`        |
| Search input               | `t.search.placeholder`        |
| Language picker dropdown   | `t.settings.language`         |

## 📱 Android Client (Flutter + Dart)

### Architecture

The Android localization system is implemented as a Flutter `ChangeNotifier`:

📄 [`lib/services/localization_service.dart`](file:///d:/AI_serach_image/image_clip_android/android/lib/services/localization_service.dart)

Key design decisions:
- **Default locale: `en`** — on first launch the app always shows English.
- **Persistence via SharedPreferences** — the user's language choice is saved and restored on subsequent launches.
- **Fallback**: If a key is missing for the selected locale, the English string is returned automatically.
- **Auto-initialization**: `LocalizationService.initializeLocales()` is called as a top-level side effect to prepopulate all 20 language entries (languages without explicit translations fall back to English with a locale tag).

### Provider Setup (`main.dart`)

```dart
runApp(
  MultiProvider(
    providers: [
      ChangeNotifierProvider(create: (_) => SyncViewModel()),
      ChangeNotifierProvider(create: (_) => LocalizationService()),
    ],
    child: const ImageClipApp(),
  ),
);
```

### Usage in Widgets

```dart
final t = Provider.of<LocalizationService>(context);

Text(t.get('linkPc'))       // "Scan QR to Connect" / "扫码连接电脑"
Text(t.get('disconnected')) // "Disconnected" / "未连接"
Text(t.get('send'))         // "Send" / "发送"
```

### Language Selector Dialog

A language selector icon (🌐) is placed in the Home Screen top bar. Tapping it opens an `AlertDialog` with a scrollable `ListView` of all 20 languages. Selecting one immediately calls `localizationService.setLanguage(code)`, which updates the `ChangeNotifier` and triggers reactive UI rebuilds across all listening widgets.

```dart
IconButton(
  icon: const Icon(Icons.language, color: Colors.white70),
  onPressed: () => _showLanguageSelector(context),
),
```
